## Existing Conventions
- The method repository is the source/template repo; the meme project is the real business/project repo.
- For the meme project, prefer a simple professionalization flow: natural-language idea → source pack → research note → next task/stop.
- Borrow only lightweight loop ideas from `-xunhuan` unless the task truly needs repeated maker/checker cycles.

## Known Risks / Things Agents Often Get Wrong
- Overcomplicating the meme project with too many agents, roles, or documents.
- Treating AI-generated conclusions as facts without source packs.
- Using Codex for vague reasoning instead of bounded execution.
- Losing the user's natural-language intent by forcing technical jargon too early.

## Decisions That Should Not Be Re-litigated
- The user wants a simple workflow that still looks professional.
- The system should support non-professional natural-language input.
- The system should emphasize external professional knowledge and evidence.
- The meme project should stay lightweight and practical rather than becoming an AI-workflow research project.
