# 02 A→B→C Loop Mechanism

## 基础流程
```text
原始想法
→ A端：保存原始意图 / 澄清目标 / 定边界
→ B端：策略化搜证 / source pack / benchmark
→ C端：理论合成 / 长文结构化 / 分级
→ 回到 A端：吸收、重搜、继续写、落库或暂缓
```

## 闸门
### A→B
- 问题已经能说清
- 需要证据补强
- 需要对标和 source pack

### B→C
- 资料足够压缩
- 证据边界明确
- 可以开始理论合成

### C→A
- 发现资料不足
- 发现边界不清
- 发现理论冲突
- 发现需要 owner 决策

## 重点
A 控制问题定义和资料吸收，不让 B 泛搜索，也不让 C 硬写理论。
