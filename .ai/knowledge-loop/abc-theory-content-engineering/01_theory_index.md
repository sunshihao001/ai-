# 01 Theory Index｜理论索引

## 索引规则
每条理论都要标注：
- 端口：A / B / C / A-B / B-C / A-B-C
- 成熟度：L0-L5
- 状态：raw / candidate / accepted / partial / needs_evidence / needs_full_text / rejected
- 推荐落点：进入哪个文件
- 下一步动作：澄清 / 补证据 / 合成 / 归档 / 暂停

## 核心理论条目
- A端是 dominant front door
- A端是 Intent-to-Spec Control Gate
- B端是 source pack 层
- B端先策略后搜索
- C端是理论合成层
- C端负责 L0-L5 分级
- dbskill 是命令家族，不是新底座
- Spec Kit 增强 A 端规格化
- Fractals 只作为后续执行参考
- X 文章仅有摘要，状态为 needs_full_text
