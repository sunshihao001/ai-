# /goal prompt｜A/B/C 知识循环体系结构

请基于 `source-pack.md` 执行 C端长理论梳理任务。

## 任务

把关于 A端 + B端 + C端 的全部理论、散的理论、半成品理论、命令映射、递归工作流想法，统一整理进“知识循环体系结构”的内容工程中。

## 边界

- 只处理知识循环体系结构层
- 不做 D/E/F 的完整执行系统设计
- 不把 Fractals / dbskill / Spec Kit 当成新底座
- 不发明平行新工作流
- 不臆造 X 文章正文，只使用已验证可见摘要

## 必须输出

1. 总览：当前理论研究边界
2. A端理论层
3. B端理论层
4. C端理论层
5. A→B→C 知识循环结构
6. 内容工程文件结构
7. 理论分级体系 L0-L5
8. dbskill / Fractals / Spec Kit 的桥接位置
9. 未成熟理论与待研究问题
10. 下一轮 C端提示词

## 重点要求

- A 端是 dominant front door
- A → B → C 是常见理论循环
- 每条理论要标注端口、成熟度、状态、推荐落点、下一步动作
- 输出必须能进入文件，不只是口头总结
- 用中文输出，适合非专业用户阅读
