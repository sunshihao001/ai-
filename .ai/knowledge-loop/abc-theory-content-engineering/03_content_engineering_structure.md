# 03 Content Engineering Structure

## 目录结构
```text
content-engineering/
  00_intent/
  10_b_port/
  20_c_port/
  30_command_mapping/
  40_review/
  50_archive/
```

## 各层职责
### 00_intent
保存 A 端产物：raw ideas、good question、goal audit、concept deconstruction、boundary。

### 10_b_port
保存 B 端产物：search strategy、source pack、benchmark notes、evidence log、rejected sources。

### 20_c_port
保存 C 端产物：theory index、theory outline、theory draft、article draft、framework map、open questions。

### 30_command_mapping
保存 dbskill / Fractals / Spec Kit 如何桥接到 A/B/C。

### 40_review
保存吸收决策、修订记录、成熟度检查、冲突。

### 50_archive
保存阶段性成品、报告、恢复锚点。

## 规则
- 内容必须文件化，不要只留在聊天里
- A 先于 B，B 服务 C
- 桥接工具不等于新底座
