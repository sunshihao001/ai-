# Extracted Content Units

## Unit list

### U1 — A-port first reading

- Read repo basics before integration.
- Start from README / install docs / AGENTS / quick start.
- Summarize the project in its own terms first.

### U2 — Install-surface-first

- Identify install / init / validate / run commands.
- Inspect generated artifacts and runtime assumptions.
- Do not treat the whole codebase as the first source of truth.

### U3 — Bridge later, not first

- Do not absorb immediately.
- Use bridge / sandbox / defer only after the surface is understood.

### U4 — Control-plane separation

- Separate install layer, execution layer, loop/control-plane layer, and governance layer.

### U5 — Finished-project absorption filter

- Mature repos need a source pack, fit report, and absorption decision before baseline updates.

### U6 — Knowledge-loop maintenance

- Keep source notes, synthesis, frame-updates, and decisions separate.
- Preserve evidence and next gaps.

### U7 — Fractals adoption pattern

- Use practical bridge + sandbox adopt candidate.
- Borrow decomposition and tree-shaped planning, not the full runtime engine.

### U8 — dbskill intake policy

- Most skill repos are skill-layer or pattern-only.
- Only a small minority become B2 finished-project work.

## Extraction rule

If a source claim does not change routing, boundary, artifact shape, or verification, keep it as context only.
