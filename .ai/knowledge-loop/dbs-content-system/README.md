# dbs-content-system

> Single-directory heavy module for the dbskill content-structuring workflow.

## Goal

Turn the dbskill / A-port theory stream into a reusable content-engineering module:

```text
audit content scale + boundary
→ copy original source material
→ extract content units
→ build topic map
→ assemble candidate drafts
→ keep evidence and next gaps in one directory
```

## Latest theory used here

- A-port first reading: read project basics before integration.
- Install-surface-first: commands and generated artifacts first, theory second.
- Bridge later, not first: do not absorb a repo before understanding its basic surface.
- Keep the module single-directory and reviewable.

## What lives here

- `00_audit.md` — content scale and boundary audit
- `01_source_pack.md` — original source material inventory and copied excerpts
- `02_content_units.md` — extracted atomic content units
- `03_topic_map.md` — topic map and grouping
- `04_assembly_drafts.md` — candidate draft assembly packages
- `99_run_report.md` — archived run log (see `.ai/knowledge-loop/archive/dbs-content-system/`)

## Source boundary

Sources used for this module should come from:

- `README.md`
- `AGENTS.md`
- `CONTEXT.md`
- `.ai/methods/*`
- `.ai/research/*`
- `.ai/knowledge-loop/*`

Avoid mixing in secondary summaries before the original material has been copied or indexed.

## Done condition

The module is mature when it can answer:

- what the source corpus contains
- what the stable content units are
- what topics should be assembled
- what remains too noisy or too broad
- which next drafts should be written first
