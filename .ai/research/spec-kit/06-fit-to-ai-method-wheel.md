# Spec Kit Fit to AI Method Wheel

## 1. Corrected insertion point

The owner's correction is:

```text
Do not jump from repo discovery to bridge decision.
First build a full project-understanding package.
```

Therefore the workflow needs a new component:

```text
B2 Finished Project Absorption
```

Position:

```text
A: asks whether an external project may improve workflow
→ B: gathers source pack / repo evidence
→ B2: understands the finished project across theory/code/scaffold/ecosystem
→ A: makes absorption decision
→ C: synthesizes bridge/spec/method update
→ D: lands bounded docs/code
→ E: verifies consistency
→ F: approves baseline changes
```

## 3. Classification

```text
ADOPT:
- artifact spine vocabulary
- WHAT/HOW separation
- requirements checklist as English unit tests
- analyze as E-port read-only gate
- task queue format principles

BRIDGE:
- constitution → F/A governance
- specify/clarify/checklist → A
- plan/research/contracts → C
- tasks → D
- implement → D only after gates
- converge → E→D repair loop

MERGE:
- presets with internal skill/template layering
- extensions with external skill governance
- workflows with A/B/C/D/E/F run records

PATTERN_ONLY:
- direct CLI installation
- Hermes integration code path
- bundle distribution model
- catalog architecture

REJECT AS BASELINE:
- direct implement from raw owner intent
- replacing A/B/C/D/E/F with linear Spec Kit workflow
- installing global Hermes skills without profile/rollback checks
```

## 5. Updated bridge rule

Old mistaken shortcut:

```text
read repo → decide bridge
```

Corrected flow:

```text
read repo → build B2 finished-project absorption pack → decide bridge → update method docs
```

This prevents shallow adoption and gives future repos a repeatable standard.
